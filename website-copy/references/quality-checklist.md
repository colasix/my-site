# Quality Checklist

Before marking workflow as complete, verify:

## Content Quality
- [ ] All sections follow Problem-First narrative
- [ ] Section Narrative Plans adhered to (each section has correct conflict/resolution)
- [ ] Page Voice Notes respected (tone matches page type)
- [ ] Tone is consistent across all pages
- [ ] No generic marketing clichés
- [ ] Trust signals included where appropriate
- [ ] CTAs are clear and compelling
- [ ] Page review completed for all wireframes (via review subagents)
- [ ] No claims contradict source documents
- [ ] **No repetition (within page):** Each major stat/differentiator appears only ONCE prominently per page
- [ ] **No repetition (across pages):** Global stats registry shows each stat featured prominently on ONE page only
- [ ] **Varied proof points:** Each section introduces NEW supporting evidence per narrative plan
- [ ] **Source docs referenced:** Each section cites specific source documents used
- [ ] **Predicate expression:** Every core predicate for this page expressed naturally in copy (check against Predicate Coverage table)
- [ ] **Frame tone:** Each section's emotional register matches its assigned frame element — protagonist sections feel empathetic, guide sections feel authoritative, antagonist sections feel urgent
- [ ] **Semantic roles clear:** Each section's copy makes agent, action, and beneficiary evident without explicit labelling
- [ ] **Transitional coherence:** Sections flow logically — each section's opening connects to the previous section's resolution, per transition annotations
- [ ] **Context depth executed:** DEEP sections contain comprehensive treatment with multiple subtopics and evidence; BROAD sections provide concise overview without overloading

## Technical Quality
- [ ] All HTML is valid and semantic
- [ ] No classes or IDs in HTML
- [ ] NZ/AU spelling throughout
- [ ] No em-dashes present
- [ ] All links properly formatted
- [ ] Section comments preserved

## SEO Quality
- [ ] Primary keywords naturally incorporated
- [ ] Semantic entities present
- [ ] Headings follow hierarchy (H2 > H3 > H4)
- [ ] No keyword stuffing
- [ ] SEO Distribution Plan followed
- [ ] Each primary keyword appears max 2x per page
- [ ] Entity clustering applied (related terms grouped)
- [ ] Location terms concentrated in appropriate sections
- [ ] Search terms from Query Vocabulary Map appear in H2 headings (not just brand terminology)
- [ ] Thematic vocabulary variety — each section uses 3-4 terms from its assigned thematic field, with lexical variety (no single term dominates)
- [ ] No key term repeated more than 2x per page without a synonym or variation intervening
- [ ] Entity structuring — named people, organisations, and services are introduced with full context on first mention (name, role, credential) for entity recognition
- [ ] Semantic bridges — cross-page internal links use natural phrasing from the Semantic Content Network, placed at annotated bridge points
